# Why Agentic Systems Need Debt Governance

Traditional software can accumulate technical debt silently for some time.
Agentic systems accumulate more than technical debt.
They accumulate:
- technical debt;
- documentary debt;
- evaluative debt;
- cognitive debt;
- governance debt.

This matters because agentic systems do not merely execute logic.
They also shape decisions, interpretations, routing, escalation and trust.

When debt remains invisible, operators may wrongly assume that a system is governed simply because it still produces outputs.
But useful output is not the same as operational integrity.

Debt governance is therefore not a secondary practice.
It is a first-class requirement for any serious agentic architecture.

A governed system must know:
- what is pending;
- what is risky;
- what is degraded;
- what is canonical;
- who owns remediation.
